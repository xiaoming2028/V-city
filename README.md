## 4. Persona 认证的三层风控机制

如果不慎触发了 Persona 实名认证，你需要理解它究竟在核验什么。Persona 的身份验证体系建立在**多维信号交叉验证**的架构上，共分三层：

### 第一层：证件真实性验证（Document Verification）

- 上传护照或身份证后，系统对证件进行 **OCR 文字识别**
- 与证件数据库进行比对核验
- 检测防伪特征（如水印、荧光墨水、微缩字体等）
- 假证或经过 Photoshop 修改的图片，在这一层直接被拦截

### 第二层：人证一致性验证（Liveness Detection + Face Matching）

- 要求用户对着摄像头完成指定的活体检测动作（如眨眼、转头）
- 将实时人脸与证件照片进行生物特征比对
- 确认"持证人"与"证件本人"是同一个人，防止冒用他人证件

### 第三层：风险环境信号综合评估（Risk Signal Aggregation）——核心暗礁

Persona 在做身份验证的同时，还会收集大量底层环境信号——IP 地址、设备指纹、注册行为模式、地理位置一致性。比如它会利用 WebRTC 穿透代理读取你真实的本地 IP，手机扫码时会读取你的 GPS 或运营商信息（如中国移动/联通）。

Persona 将上述信号汇总后生成一个**风险评分**，提交给 Anthropic 进行最终裁决。得分过高，即便你的证件是真实的，也可能被系统拒绝。

**目前观察到的触发规律**：Persona 认证不会对所有用户普发，主要在以下情况下触发：
- 账号触碰高危功能（如大规模 API 调用）
- 发生明显的异常行为（如网络环境突然跳变）
- 系统判定账号风险得分超过阈值
